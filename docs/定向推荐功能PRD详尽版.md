## 一、项目背景

### 1.1 需求背景
当前APP【发现】模块支持6大类别（资产推介、竞拍服务商、资金方、明星经理、专项招募等），现需新增**"定向推荐"**类型，实现对特定人群的精准项目推送。

### 1.2 核心价值
- **精准触达**：定向推送给特定合营服务商，而非全量用户
- **信息隔离**：通过白名单机制控制可见范围
- **商业闭环**：业务团队发布 → 领导审批 → 系统推送 → 目标用户查看 → 线下对接

### 1.3 目标用户
| 角色 | 职责 | 使用场景 |
|------|------|---------|
| 业务团队 | 录入项目信息、上传附件、配置白名单 | 创建定向推荐项目 |
| 领导（于总） | 审核项目信息 | 审批发布请求 |
| 运营人员 | 查看列表、操作撤回/下架 | 运营中心管理 |
| 目标用户（合营服务商） | 查看推荐项目 | APP/小程序/H5查看 |

## 三、详细功能设计

### 3.1 发现配置管理（运营中心）

#### 3.1.1 功能说明
在运营中心的【发现配置】中新增"定向推荐"类型，支持置顶排序。

#### 3.1.2 配置项
| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 类型名称 | 文本 | 是 | 默认"定向推荐" |
| 类型编码 | 文本 | 是 | 默认"direct_recommend" |
| 是否启用 | 开关 | 是 | 控制该类型是否在前端展示 |
| 排序权重 | 数字 | 是 | 数值越大越靠前，新类型默认最高 |
| 是否置顶 | 开关 | 是 | 开启后在发现列表置顶展示 |

#### 3.3.2 字段详细说明

| 项目编号 | 文本 | 系统自动 | 只读，格式：DR+年月日+4位序号 | 系统自动生成，不可编辑 |
| 资产名称 | 文本 | 是 | 长度2-200字符 | 必填提示：请输入资产名称，如"广州白云核心区住宅项目" |
| 资产分类 | 下拉单选 | 是 | 关联字典表 | 选项：住宅地产、商业地产、工业地产、不良资产、股权投资、其他 |

| 经度 | 数字 | 否 | -180~180 | 地图选点自动填充 |

| 字段名 | 类型 | 必填 | 校验规则 | 交互说明 |

**上传后处理流程：**
1. 用户上传PDF文件
2. 系统校验格式和大小
3. 系统自动添加满屏水印（水印规则见下方）
4. 系统将PDF转换为长图（用于移动端展示）
5. 存储原始PDF和水印长图两个版本

**水印规则：**
- 已实名用户：手机号 + 姓名
- 未实名用户：手机号 + 用户ID
- 水印样式：满屏半透明斜向文字，防截图泄露

#### 3.3.3 表单校验规则汇总

**必填校验：**
- 资产名称、资产分类、所在地区、资金类型、资金规模、合作期限、项目附件、发布机构、发布人员、联系电话、展示期限、白名单人员

**格式校验：**
- 手机号：11位数字，正则验证
- 日期：开始日期≤结束日期，开始日期≥今天
- 金额：最大值≥最小值，≥0
- 文件：仅PDF格式，≤50MB

**业务校验：**
- 白名单手机号去重（同一项目内不能重复）
- 联系人至少1个，最多5个
- 附件必须上传且处理成功

| 提交审批 | 右上角 | 校验全部字段，提交给领导审批 | 所有必填字段通过校验 |
| 预览 | 右上角 | 预览对外展示效果 | 随时可点击 |
| 取消 | 底部 | 返回列表页，不保存 | 有未保存内容时提示确认 |

**提交后流程：**
1. 前端校验全部字段
2. 校验通过，状态变为"待审批"
3. 系统通知领导（于总）审批
4. 锁定表单，不可编辑（仅可预览、撤回）

## 六、推送机制

### 6.1 触发条件
项目状态变为"已发布"时触发推送。

### 6.2 推送渠道
| 渠道 | 内容 | 说明 |
|------|------|------|
| 短信 | "【XX平台】您有一条新的项目推荐，请点击查看：短链" | 发送到白名单手机号 |
| APP推送 | "有新的定向推荐项目，点击查看详情" | 已注册用户 |
| 站内信 | 消息中心展示 | 登录后可见 |

### 6.3 短链逻辑
```
短链 -> H5发现列表 -> 登录校验 -> 白名单校验 -> 展示详情
```

**安全控制：**
- 短链有效期：与项目展示期限一致
- 转发控制：他人打开需登录+白名单校验
- 登录校验：使用当前登录账号校验是否在白名单
