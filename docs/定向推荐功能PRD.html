一、项目背景

1.1 需求背景

当前APP【发现】模块支持6大类别（资产推介、竞拍服务商、资金方、明星经理、专项招募等），现需新增"定向推荐"类型，实现对特定人群的精准项目推送。

1.2 核心价值

  • 精准触达：定向推送给特定合营服务商，而非全量用户
  • 信息隔离：通过白名单机制控制可见范围
  • 商业闭环：业务团队发布 → 领导审批 → 系统推送 → 目标用户查看 → 线下对接

1.3 目标用户

角色 职责 使用场景
业务团队 录入项目信息、上传附件、配置白名单 创建定向推荐项目
领导（于总） 审核项目信息 审批发布请求
运营人员 查看列表、操作撤回/下架 运营中心管理
目标用户（合营服务商） 查看推荐项目 APP/小程序/H5查看

3.3.2 字段详细说明

Step 1: 基本信息
字段名 类型 必填 校验规则 交互说明
项目编号 文本 系统自动 只读，格式：DR+年月日+4位序号 系统自动生成，不可编辑
资产名称 文本 是 长度2-200字符 必填提示：请输入资产名称，如"广州白云核心区住宅项目"
资产分类 下拉单选 是 关联字典表 选项：住宅地产、商业地产、工业地产、不良资产、股权投资、其他
所在地区 级联选择 是 省市区三级联动 使用国标编码，支持搜索
详细位置 文本 否 长度0-500字符 提示：详细地址或位置描述，如"白云新城核心区，近地铁2号线"
经度 数字 否 -180~180 地图选点自动填充
纬度 数字 否 -90~90 地图选点自动填充
Step 2: 资产详情
字段名 类型 必填 校验规则 交互说明
资产简介 多行文本 是 长度10-2000字符 必填提示：简要描述资产核心卖点、项目背景
项目配套 多行文本 否 长度0-2000字符 提示：周边配套资源，如凯悦酒店、龙湖代建、山湖景观等
市场分析 多行文本 否 长度0-2000字符 提示：区域供需情况、竞争优势、目标客群等
核心亮点 多行文本 否 长度0-1000字符 提示：劣后投资人储备充足、回款预期较好等
风控措施 多行文本 否 长度0-1000字符 提示：抵质押措施、担保情况、还款来源等
Step 3: 资金需求
字段名 类型 必填 校验规则 交互说明
资金类型 下拉多选 是 至少选1项 选项：夹层、劣后、优先、平层、可转债、其他
资金规模-最小值 数字 条件 若填写则≥0 与最大值组成区间
资金规模-最大值 数字 是 必须≥最小值 单位选择：万元/亿元
合作期限 数字 是 整数，≥1 单位选择：年/月/季度
资金用途 多行文本 否 长度0-500字符 提示：具体资金用途说明
预期收益 文本 否 长度0-100字符 提示：如"年化8%-12%"
Step 4: 项目附件
字段名 类型 必填 校验规则 交互说明
项目附件 文件上传 是 仅PDF，≤50MB 支持拖拽上传，上传后显示文件名和大小
附件预览 预览组件 - - 点击预览按钮，显示PDF转长图后的效果
水印说明 提示文本 - - 上传后系统自动添加水印（手机号+姓名）

上传后处理流程： 1. 用户上传PDF文件 2. 系统校验格式和大小 3. 系统自动添加满屏水印（水印规则见下方） 4. 系统将PDF转换为长图（用于移动端展示） 5. 存储原始PDF和水印长图两个版本

水印规则： - 已实名用户：手机号 + 姓名 - 未实名用户：手机号 + 用户ID - 水印样式：满屏半透明斜向文字，防截图泄露

Step 5: 发布设置
字段名 类型 必填 校验规则 交互说明
发布机构 文本 是 自动填充 自动填充当前用户所属机构，可修改
发布人员 文本 是 自动填充 自动填充当前用户姓名，只读
联系电话 手机 是 11位手机号 自动填充当前用户手机号，可修改
展示期限-开始 日期 是 默认当天 日期选择器，不能早于今天
展示期限-结束 日期 是 必须≥开始日期 默认开始日期+30天
是否置顶 开关 是 默认关闭 开启后在发现列表置顶展示
Step 6: 项目联系人（可添加多人）
字段名 类型 必填 校验规则 交互说明
联系人姓名 文本 是 长度2-50字符 -
联系人称谓 下拉单选 是 - 选项：项目经理、业务总监、总经理、实控人、其他
联系人手机 手机 是 11位手机号 必填，用于目标用户联系
联系人微信 文本 否 长度0-50字符 选填
首要联系人 单选 是 只能选一个 标识该联系人为主要对接人

交互说明： - 至少添加1个联系人 - 最多添加5个联系人 - 可拖拽调整联系人展示顺序

Step 7: 白名单配置
字段名 类型 必填 校验规则 交互说明
白名单人员 表格 是 至少1人 支持批量导入和手动添加

批量导入： - 支持Excel导入 - 模板下载：[下载Excel模板] - 必填字段：机构名称、联系人姓名、手机号 - 选填字段：机构简称 - 导入后校验手机号格式，错误行高亮提示

手动添加： - 表单字段：机构名称、联系人姓名、手机号 - 添加后实时显示在列表中 - 支持删除单条记录

白名单列表展示字段： | 字段 | 说明 | |------|------| | 序号 | 自动编号 | | 机构名称 | 导入或手动输入的机构全称 | | 联系人姓名 | - | | 手机号 | 脱敏展示，如138****1234 | | 系统注册状态 | 已注册/未注册（自动识别） | | 操作 | 删除、编辑 |

3.3.3 表单校验规则汇总

必填校验： - 资产名称、资产分类、所在地区、资金类型、资金规模、合作期限、项目附件、发布机构、发布人员、联系电话、展示期限、白名单人员

格式校验： - 手机号：11位数字，正则验证 - 日期：开始日期≤结束日期，开始日期≥今天 - 金额：最大值≥最小值，≥0 - 文件：仅PDF格式，≤50MB

业务校验： - 白名单手机号去重（同一项目内不能重复） - 联系人至少1个，最多5个 - 附件必须上传且处理成功

3.3.4 按钮与操作

按钮 位置 功能 触发条件
保存草稿 右上角 保存当前填写内容，状态变为"草稿" 随时可点击
提交审批 右上角 校验全部字段，提交给领导审批 所有必填字段通过校验
预览 右上角 预览对外展示效果 随时可点击
取消 底部 返回列表页，不保存 有未保存内容时提示确认

提交后流程： 1. 前端校验全部字段 2. 校验通过，状态变为"待审批" 3. 系统通知领导（于总）审批 4. 锁定表单，不可编辑（仅可预览、撤回）

  • 领导：看到全部待审批项目

  • 普通用户：只能看到自己在白名单中的项目

六、推送机制

6.1 触发条件

项目状态变为"已发布"时触发推送。

6.2 推送渠道

渠道 内容 说明
短信 "【XX平台】您有一条新的项目推荐，请点击查看：短链" 发送到白名单手机号
APP推送 "有新的定向推荐项目，点击查看详情" 已注册用户
站内信 消息中心展示 登录后可见

6.3 短链逻辑

短链 -> H5发现列表 -> 登录校验 -> 白名单校验 -> 展示详情

安全控制： - 短链有效期：与项目展示期限一致 - 转发控制：他人打开需登录+白名单校验 - 登录校验：使用当前登录账号校验是否在白名单

七、非功能需求

7.1 性能要求

  • 列表页加载：≤2秒（100条数据）
  • 详情页加载：≤1秒
  • 文件上传：≤30秒（50MB文件）
  • PDF转长图：≤60秒（后台异步处理）

7.2 兼容性

  • 后台：Chrome、Firefox、Safari最新2个版本
  • APP：iOS 12+、Android 8+
  • 小程序：微信、支付宝、抖音基础库2.0+

7.3 安全要求

  • 手机号加密存储（AES）
  • PDF加水印（满屏半透明）
  • 白名单访问控制
  • 操作日志记录

7.4 备份与恢复

  • 项目数据每日备份
  • 审批记录永久保留
  • 支持误删恢复（30天内）
